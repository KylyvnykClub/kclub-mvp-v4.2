# GitNexus Workflow

GitNexus is optional context acceleration, not an authority over live files, tests, or documentation. When available, re-index after structural changes and use impact analysis before changing shared contracts, domain policies, UI primitives, database schema, or cross-module flows.

```text
analyze -> inspect context/impact -> make scoped change -> detect changes -> cycle check
```

Record material ownership changes in an ADR. If GitNexus is unavailable or stale, inspect the live dependency graph and tests directly; do not block implementation or invent findings.
